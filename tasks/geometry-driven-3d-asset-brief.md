Move away from character-led 3D thinking and toward geometry-driven 3D assets inspired by the existing witnessOS visual system.

This is not a small stylistic adjustment. It is a conceptual reset:

- from `beings`
- to `instruments`

- from `figures`
- to `geometric apparatus`

- from `character identity`
- to `stateful symbolic computation`

## What The Images Actually Suggest

### 1. This system is planar, radial, and etched

The references are dominated by:

- concentric rings
- lotus / petal symmetries
- radial compass axes
- waveform overlays
- thin-node constellations
- rectangular or octagonal container frames
- measured panel systems

This is not character concept art language.

It is:

- interface cosmology
- sacred instrumentation
- engraved signal geometry

### 2. The most successful forms are 2.5D, not volumetric sculptures

The references do not want chunky statues.
They want:

- relief depth
- layered ring assemblies
- recessed channels
- luminous linework
- glass/metal panel stacks
- medallions, dials, apertures, shells, and cartographic plates

So the right 3D move is usually:

- shallow depth
- layered geometry
- bevel + glow + parallax

Not:

- free-standing creatures
- anthropomorphic silhouettes
- museum busts

### 3. The system language is stateful

These visuals encode:

- coherence states
- breath stages
- opportunity windows
- directional decisions
- correlations
- wave signatures

So a 3D object should not just look geometric.
It should have a state logic:

- ring opens
- wave brightens
- node connects
- axis turns
- chamber fills
- field stabilizes

### 4. Geometry is the narrative

The strongest reference screens already make the case:

- `breathnav-screen` turns a breathing interaction into concentric-action geometry
- `hrv` turns coherence into waveform + mandala coupling
- `portfolio` turns financial state into a radial field map
- `dashboard` turns insights and signals into a panelized observatory

That means the landing-page 3D assets should not be illustrations of engines.
They should be geometry-native manifestations of engine logic.

## Replacement Principle

The landing page should evolve from:

- `poster as image`

to:

- `poster as living geometry object`

Each section still keeps its 1:1 identity, but the object class should be derived from the witnessOS geometry system rather than from character or creature logic.

## What To Stop Doing

Do not use:

- humanoid stance
- ribs, shoulders, hands, limbs
- faceplates implying a head
- fantasy relic characters
- body-shaped silhouettes as the default answer

The witnessOS references simply do not support that move.

## What To Start Doing

For future Meshy passes, the opening phrase should usually be one of:

- `radial instrument`
- `concentric geometry object`
- `breath-state dial`
- `coherence gate`
- `waveform mandala disc`
- `field ring apparatus`
- `node lattice plate`
- `compass-frame aperture`

That is much closer to the actual brand language in these references than:

- `character`
- `vessel being`
- `torso`
- `guardian`
- `sentinel`

## Best Immediate Pivot

If we use these images seriously, the product-level conclusion is:

- Aletheios and Pichet do not need to be characters
- they can be poles expressed through distinct geometry systems

Suggested split:

- `Aletheios`
  - dial / compass / observatory geometry
  - gold-led
  - exact, radial, calibrated

- `Pichet`
  - breath / wave / coherence geometry
  - emerald and violet-led
  - pulsing, chambered, stateful

That preserves the dyad without turning it into two people.

## Recommended Next Steps

1. Rewrite the phase-one object prompts using witnessOS geometry vocabulary only.
2. Stop using body nouns for `Pichet` entirely.
3. Recast `Aletheios` as a suspended compass-dial instrument rather than a reliquary object on a stand.
4. Build one prompt set for `radial instruments` and one for `signal fields` so the family resemblance is coherent.
5. Only after the geometry language is stable should any landing-page embedding or model optimization work resume.

## Bottom Line

Yes: moving away from character is the right call.

These references make the answer much more specific:

the 3D system should be geometry-driven, panel-native, radial, wave-based, and stateful.

Not figurative.
Not mascot-led.
Not body-first.

The landing page should feel like a ritual observatory made of living instruments, not a world populated by symbolic beings.
