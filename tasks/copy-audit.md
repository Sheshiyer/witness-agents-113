# Witness Agents Copy Audit

Date: 2026-04-28
Project: `witness-agents-intro-web`
Scope: public-facing copy review before any copy edits

## Canonical Read

### Public corpus reviewed
- `Synchronocities` routes: `/`, `/research`, `/journeys`, `/maps`
- 28 unique post pages linked from those routes
- Key anchor pages:
  - `The Source Code Has Authors`
  - `An Introduction to Tryambakam Noesis: Rewrite Your Operating System`

### High-level public narrative actually visible today
The public corpus currently reads as a research-and-journey field with three strong modes:
- `research`: technical-mystical essays, references, and hubs
- `journeys`: autobiographical/travel/tarot runtime essays
- `maps`: orientation and synthesis nodes

That corpus does not read like a launched public SaaS, consumer app, or fully operational agent product. It reads like an open research ecosystem with some live artifacts and some internal architecture exposed by implication.

## Core Diagnosis

The landing page currently overstates productization.

It frames `Witness Agents` as if the public can already access a stable, user-ready interpretation layer with:
- named agent roles
- tiered access architecture
- telemetry
- subscriber flows
- enterprise inquiry flow
- invite-only identity layer
- portable multi-LLM deployment
- a functioning dispatch/list workflow

Some of those may exist as internal architecture, roadmap, or research language. They are not established cleanly enough on the public surface to be presented as settled public product reality.

## What Seems Publicly Supported
- Tryambakam Noesis as a broader field/research program
- Selemene as a live engine/API surface
- Somatic Canticles / `1319` as a live reading/canticle surface
- Open-source repos as proof-bearing artifacts
- Kha-Ba-La as governing grammar
- Self-consciousness / authorship / anti-dependency as core philosophical frame
- Lineage/influence language, if it remains specific and sourced

## What Is Not Safely Supported On The Current Public Surface
- `kopina.io` as a live tapestry node for this ecosystem
- `144` / admin as a public-facing node label
- Witness Agents as a public-ready product with live user pathways
- dyadic agent operations as a stable, externally usable feature set
- PETRAE / AKSHARA as publicly legible product mechanisms without explanation or proof
- telemetry / anti-dependency tracking claims as operational product behavior
- newsletter subscription as a live trustworthy workflow
- tiered kosha access architecture as an actual public offer stack

## Attribution Policy Recommendation

### Keep public
- Intellectual lineage that maps clearly to visible system concepts
- Named authors when the mapping is concrete and legible
- Source-conscious phrasing such as “this work stands on” or “influences / lineage / sources”

### Keep, but reframe carefully
- Claims that “every source is named” should only remain if the site actually names the relevant sources on-page or links to a fuller citation route
- A compressed lineage section is viable, but it should avoid implying exhaustive attribution if it is selective

### Do not move onto this landing page
- The personal relational credits from `The Source Code Has Authors` (partner, business partner, parents) should remain in the essay unless there is an explicit decision to make that personal credit architecture part of this page
- Highly internal implementation language that only makes sense to someone already inside the system (`AKSHARA mirror`, `identity clause`, `universal-prana`, `Clifford-gated depth`) should not be used as public primary copy without grounding

### Suggested public attribution model
- Landing page: concise `Lineage / Sources / What this work stands on`
- Essay layer: full authorship and contribution architecture
- Optional footer/link: `Read the lineage essay`

## Section-by-Section Audit

### 1. Hero
Status: `rewrite`

Current issue:
- Strong and mostly on-brand, but it positions the site as a productized agent layer rather than a research/introduction node.
- `Two agents sitting between sixteen engines and the practitioner who reads them` is stronger than the public evidence supports.

Keep:
- `Self-consciousness as technology`
- `Body as medium. Breath as interface.`
- open-source proof gestures

Change:
- Reframe hero from `agent product` to `research/practice architecture`
- Keep `1319` CTA only if the canticle remains the most honest public doorway

### 2. Why / Invariants cards
Status: `rewrite`

Current issue:
- Several claims present internal mechanism as public fact: telemetry, dyad operations, PETRAE compression, AKSHARA seeding, universal portability.

Recommendation:
- Convert to principles rather than implementation claims
- Replace unprovable product behavior with verifiable philosophical constraints

### 3. Versus strip
Status: `partial rewrite`

Current issue:
- The framing is strong, but it still assumes a public product stance.
- Uses `productivity`, which the brand docs themselves flag as avoid-language.
- Heading uses `path`, also on the avoid list.

Recommendation:
- Keep the contrast logic
- Remove self-contradicting banned vocabulary
- Shift from `this product versus others` to `this inquiry versus adjacent modes`

### 4. Core WITNESSED article sections
Status: `mixed`

Keep with light revision:
- The conceptual sections around authorship, interpretation, witness, and the gap between signal and self

Rewrite/remove:
- Any statement that turns the agent layer into a live public service promise
- Any engine-specific statement that cannot be defended from the public surface

### 5. Dyad naming sections (`Aletheios`, `Pichet`)
Status: `rewrite heavily` or `remove from primary landing flow`

Current issue:
- These sections are written as if the dyad is already a stable public interpretive interface.
- The names may exist canonically, but the current presentation reads like shipping feature copy.

Recommendation:
- Either recast them as conceptual poles within the practice
- Or move them deeper into a secondary section / essay, not the core landing conversion flow

### 6. Engine / kosha architecture sections
Status: `rewrite`

Current issue:
- `What is paid for is depth, not access` and the kosha/tier architecture implies an actual commercial offer structure.
- Subscriber / enterprise / invite-only language is not safely public.

Recommendation:
- Separate philosophical kosha framing from offer/tier language
- If there is no public product stack, do not simulate one

### 7. Tapestry nodes
Status: `rewrite immediately`

Current issue:
- Publicly exposes `144°`
- Publicly links `kopina.io`
- Mixes public nodes, private/admin nodes, easter eggs, and external repos without a clean public-information boundary

Recommendation:
- Keep only genuinely public, current nodes
- Strong likely replacements: `research`, `journeys`, `maps`, `1319`, `selemene`, repo(s), `tryambakam`, maybe `113`

### 8. Threshold / kosha vows
Status: `rewrite immediately`

Current issue:
- This is the strongest false-productization block on the page.
- It creates a quasi-pricing/offer/access ladder without a confirmed public offer stack.

Recommendation:
- Replace with honest public entry points
- Example frame: `Read`, `Explore`, `Run`, `Correspond`
- If no subscriber system exists, remove subscriber-tier language entirely

### 9. Lineage section
Status: `keep, but expand/reframe`

Current issue:
- The section is directionally correct, but too compressed relative to the actual lineage architecture in `The Source Code Has Authors`.
- `Every claim in the system traces to source documents — and every source is named` overpromises unless citation support is visible.

Recommendation:
- Keep named influences
- Tighten the promise
- Possibly add a route to the full lineage essay
- Consider whether the current list should better align with the deeper essay authors (Tsarion, James True, Gebser, Whitehead, Sheldrake, Rank, Passio, Montalk, Mead, Keelen, Slick Dissident, etc.) or whether this page should deliberately stay short and foundational

### 10. FAQ
Status: `rewrite heavily`

Current issue:
- The FAQ contains the densest cluster of unsupported implementation claims:
  - agents traversing structured knowledge graphs across any LLM
  - PETRAE conversational layer
  - anti-dependency telemetry
  - portable agent layer
  - user-ready interpretive operations

Recommendation:
- Rewrite around public truths only
- Keep the consciousness vs self-consciousness distinction
- Avoid backend/ops claims unless they are defensible and visible

### 11. Newsletter / dispatch form
Status: `rewrite or disable`

Current issue:
- The copy promises a live dispatch rhythm.
- The implementation posts to a placeholder endpoint and fakes success fallback.

Recommendation:
- Do not imply a live subscription flow if one is not operating
- Options:
  - replace with explicit email contact
  - replace with `dispatch not yet public`
  - or wire a real endpoint before keeping this section

### 12. Footer / docs / maintenance copy
Status: `rewrite`

Current issue:
- Footer still links `kopina.io`
- README still describes the page as a `Witness Agents product`
- `linkMap.js` still includes admin and kopina as canonical inward-funnel destinations

Recommendation:
- Align internal docs with the corrected public positioning
- Remove private and non-canonical nodes from the runtime map

## Immediate High-Risk Items
- `kopina.io` exposure
- `144°` admin exposure
- fake/placeholder newsletter flow
- subscriber / enterprise / initiate pseudo-offers
- FAQ implementation claims that exceed public proof
- docs that keep reinforcing the wrong product frame

## Recommended Rewrite Direction

Reposition the site as:
- an introduction to the Tryambakam Noesis field
- a public orientation node for the research/practice architecture
- a bridge to live artifacts (`1319`, `selemene`, repositories, research corpus)
- a source-conscious, non-overclaiming explanation of the witness layer

Not as:
- a launched public product with stable user funnels, tiers, and agent operations

## Proposed Public Entry Model
Replace the current threshold section with 3-4 honest doors:
- `Read the corpus`
- `Enter the canticle`
- `Run the engines`
- `Correspond directly`
